High-level architecture:
    ESP32 (BLE IMU streamer)
        ↓
    Mobile App
        ↓ HTTPS
    FastAPI Backend
        ├─ API routes
        ├─ services
        ├─ analytics pipeline
        ├─ DB access
        └─ background jobs (later if needed)
        ↓
    Database

api/routes
    Only HTTP concerns.
    validate request
    call service
    return response

services
    This is the “brain of the backend flow”.
    Application/business orchestration.
    Examples:
        create workout session
        ingest sample batch
        finalize session
        trigger analytics
        fetch rep results

    session_service.py
        create session
        transition session states
        validate ownership
        finalize session
        attach exercise/device/calibration info

    ingestion_service.py
        validate incoming sample chunk
        enforce monotonic sample_index
        enforce valid sensor ranges if desired
        bulk insert samples
        update sample_count

    analytics_service.py
        load raw session samples
        run analytics.pipeline.run_session_analysis(...)
        write reps/metrics/session summary
        update session status

    calibration_service.py
        create calibration profiles
        compute/validate device offsets
        select active calibration profile        

analytics
    Pure signal-processing / metric logic.
    Keep this layer mostly independent from FastAPI and DB.
    Examples:
        filtering IMU
        detect rep boundaries
        detect concentric/eccentric phases
        compute velocity proxies
        compute shaking score
        estimate sticking point

db/models
    Database tables / ORM models.




Session pipeline
    A. Device registration / pairing phase
        User creates account
        ↓
        User registers device
        ↓
        App links device to user
        ↓
        Calibration profile may be created

    B. Workout start
        App starts session
        ↓
        POST /sessions
        ↓
        Backend creates workout_session row
        ↓
        returns session_id

    C. Live ingestion
        ESP32 streams IMU packets to app over BLE
        ↓
        App batches packets
        ↓
        POST /sessions/{session_id}/samples
        ↓
        Backend stores raw samples

        Recommended: send in chunks, not one HTTP request per sample.
            For example:
                25 samples
                50 samples
                100 samples per request

    D. Workout finalize
        App stops session
        ↓
        POST /sessions/{session_id}/finalize
        ↓
        Backend marks session as uploaded/closed
        ↓
        Analytics pipeline runs
        ↓
        reps + metrics stored
        ↓
        session summary updated

    E. Results fetch
        App requests analyzed session
        ↓
        GET /sessions/{session_id}
        GET /sessions/{session_id}/reps
        GET /sessions/{session_id}/analytics

        device_name
        hardware_model

        status

        created_at

        profile_name

        accel_bias_y

        gyro_bias_x
        gyro_bias_y
        gyro_bias_z
        orientation_reference_json
        created_at
        is_active

        id

        description

        default_axis_hint
        created_at

    5. workout_sessions - Main top-level session.
